# ChatBridge コントリビューションガイド

このドキュメントは、ChatBridge プロジェクトの開発規約を定義する。
AI ツール（Antigravity, Claude Code 等）および人間の開発者が共通して参照する。

> **📌 管理元**: このファイルの内容は Antigravity のグローバルルール（`GEMINI.md` §6）に基づく。
> グローバルルールが更新された場合は、このファイルも同期して更新される。

---

## ドキュメント管理

### ユーザー向け / 開発者向けの分離

プロジェクトのドキュメントは **ユーザー向け** と **開発者向け** を明確に分離する。

| 配置場所 | 対象読者 | 内容 |
|---|---|---|
| ルート直下 | ユーザー | README, CHANGELOG, LICENSE, CONTRIBUTING |
| `docs/` フォルダ | 開発者 | 設計、仕様、タスク、作業記録等 |

### 命名規則

- ファイル名は **大文字スネークケース**（例: `ARCHITECTURE.md`）
- **略語は使わず、フルネーム** で命名する（例: ✅ `SPECIFICATION.md` ❌ `SPEC.md`）

### 標準ドキュメント構成

```
(ルート)                        ← ユーザー向け
├── README.md                   # 概要・動機・使い方
├── CHANGELOG.md                # リリースごとの変更履歴
├── CONTRIBUTING.md             # 開発規約（本ファイル）
├── LICENSE
docs/                           ← 開発者向け
├── ARCHITECTURE.md             # 設計思想・構成・スレッドモデル
├── SPECIFICATION.md            # 機能仕様・動作仕様
├── ROADMAP.md                  # 開発計画・ビジョン
├── TASKS.md                    # 進行中のタスクリスト
├── DEVELOPMENT_LOG.md          # 完了した作業の記録
└── TROUBLESHOOTING.md          # デバッグ・問題対処（必要に応じて）
```

### ドキュメント間リンク

- ドキュメント間のリンクは**相対パス**で記述する
- ファイルを移動した場合は、リンクも必ず更新する

---

## コーディング規約

- コメントは可能な限り**日本語**で記述する
- コードは簡潔で可読性の高いものを目指す
